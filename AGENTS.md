# AGENTS.md — algorithm-practice

> 本文件供 AI 编程助手阅读。阅读者应假定对该项目一无所知。

## 项目概述

这是一个**个人算法竞赛训练仓库**，主要使用 **C++** 编写，用于记录和练习来自多个在线评测平台（OJ）的算法题目。代码以单文件形式组织，每个 `.cpp` 文件对应一道题目的完整解答。

项目没有使用任何构建系统（如 CMake、Makefile、npm、Cargo 等），所有代码均为独立的单文件程序，直接通过编译器命令编译运行。

## 技术栈

- **语言**：C++（使用 GCC/MinGW 编译器）
- **标准头文件**：所有源文件统一使用 `#include <bits/stdc++.h>`（GCC 特有的万能头文件）
- **命名空间**：所有源文件均使用 `using namespace std;`
- **编译器**：GNU G++（Windows 环境下为 MinGW-w64）
- **无外部依赖**：不依赖任何第三方库

## 编译与运行

### 编译命令

由于无构建系统，使用编译器直接编译：

```bash
# Linux / macOS
g++ -std=c++17 -O2 -o output/filename filename.cpp

# Windows (MinGW)
g++ -std=c++17 -O2 -o output\filename.exe filename.cpp
```

### 常用编译选项

- `-std=c++17`：使用 C++17 标准（代码中大量使用 `auto`、`vector` 初始化列表等现代特性）
- `-O2`：开启二级优化（竞赛常用）
- `-Wall -Wextra`：建议开启以捕获潜在警告（当前项目中未强制要求）

### 运行方式

```bash
# Linux / macOS
./output/filename < input.txt

# Windows
.\output\filename.exe < input.txt
```

## 代码风格指南

### 强制/常见约定

1. **头文件**：所有文件**必须**以如下两行开头：
   ```cpp
   #include <bits/stdc++.h>
   using namespace std;
   ```

2. **主函数**：所有可执行程序均包含 `int main(){ ... return 0; }`

3. **输入输出混合风格**：
   - 简单题目使用 `scanf` / `printf`（C 风格，追求速度）
   - 部分题目使用 `cin` / `cout`（C++ 风格，配合 `fixed << setprecision()` 处理浮点输出）
   - 同一文件中很少混用两种风格，但不同文件之间风格不统一

4. **数组索引**：
   - 传统数组/vector 常使用 **1-based 索引**（`for(int i = 1; i <= n; i++)`），这是竞赛编程常见习惯
   - 现代 C++ 容器（如 `vector`）有时也使用 0-based 索引

5. **变量命名**：
   - 使用极短变量名：`n`, `m`, `t`, `ans`, `res`, `vec`, `st`, `vis`
   - 常量使用全大写：`N`, `INF`, `EPS`

6. **注释语言**：
   - 注释以**中文**为主
   - 部分文件包含详细的解题思路注释（变量含义、算法步骤）
   - 部分文件在末尾以块注释形式保留**测试用例数据**

7. **多解法组织**：少数文件（如 `LuoGu/1.1/P1094.cpp`）使用 `namespace Solution1`、`namespace Solution2` 封装不同解法，在 `main()` 中选择调用

### 不推荐但存在的风格

- 部分文件缺少头文件与 `main()` 之间的空行分隔
- 部分文件的大括号风格不统一（K&R 风格与 Allman 风格混用）
- `return ;` 语句后有无意义的分号或空格（`return ;` vs `return;`）

## 测试策略

- **无自动化测试框架**：本项目没有单元测试、集成测试或 CI/CD 流程
- **测试方式**：
  1. 手动编译运行
  2. 输入样例数据（部分题目在文件末尾注释中保留了样例）
  3. 与平台预期输出比对
- **可执行文件目录**：编译输出常放在同级 `output/` 子目录下，或直接在平台目录中生成 `.exe`（Windows）

## 版本控制

- 使用 **Git** 管理
- 提交信息语言：中文为主，夹杂英文技术词汇
- `.gitignore` 已配置忽略编译产物（`.exe`, `.o`, `output/` 等）、IDE 配置文件（`.vscode/`, `.idea/`）及系统文件

## 安全与注意事项

- **无安全敏感代码**：本项目为纯算法竞赛练习，不涉及网络请求、文件系统敏感操作、用户认证等
- **输入假设**：所有程序均假设输入严格符合题目描述，无输入校验
- **数据范围**：常量 `N` 通常开得较大（如 `1005`, `1e5+5` 等），需注意栈溢出风险（尤其在递归深度较大时）
- **随机数使用**：部分题目（如计算几何）使用 `mt19937` 固定种子（如 `19980731`），这是竞赛中常见的确定性随机化技巧

## 对 AI 助手的建议

1. **新增题目**：在对应平台目录下新建 `.cpp` 文件，使用标准头文件模板
2. **修改代码**：保持原有注释语言（中文）和变量命名风格（简短）
3. **不要引入构建系统**：保持单文件独立编译的简单性
4. **保留测试数据**：如原文件末尾有注释形式的样例数据，修改时应尽量保留
5. **编译验证**：修改后建议使用 `g++ -std=c++17 -O2` 编译验证无语法错误
